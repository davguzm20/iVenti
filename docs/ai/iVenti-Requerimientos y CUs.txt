1. Introducción 
1.1 Propósito 
Este documento tiene como propósito definir los requisitos funcionales y no funcionales del 
sistema iVenti, una aplicación móvil orientada a la gestión de inventario y ventas para el 
negocio Multiservicios Golden S.A. Se describe el comportamiento esperado del sistema, sus 
principales funcionalidades, restricciones y características generales. 
1.2 Alcance del sistema 
El sistema permite registrar y controlar productos, gestionar ventas al contado y a crédito, 
administrar clientes deudores, generar reportes y emitir alertas relacionadas al estado del 
inventario. Todas las operaciones principales se ejecutan en modo offline, utilizando 
almacenamiento local en el dispositivo. 
1.3 Definiciones, siglas y abreviaturas 
●​ Sistema iVenti - Aplicación móvil para la gestión de inventario y ventas. 
●​ Inventario - Conjunto de productos registrados con su stock disponible. 
●​ Stock - Cantidad disponible de un producto en el inventario. 
●​ Lote - Registro de ingreso de productos con atributos como cantidad, precio de 
compra y fecha de vencimiento. 
●​ Venta al contado - Transacción pagada en su totalidad en el momento de la compra. 
●​ Venta a crédito - Transacción en la que el cliente adquiere una deuda pendiente. 
●​ Cliente deudor - Cliente con saldo pendiente por ventas a crédito. 
●​ Boleta - Documento generado que detalla una venta realizada. 
●​ PIN - Código numérico de 6 dígitos utilizado para autenticación del usuario. 
2. Descripción General 
2.1 Perspectiva del sistema 
El sistema iVenti es una aplicación móvil independiente que implementa los requisitos 
definidos a nivel de negocio y de usuario, proporcionando funcionalidades concretas para la 
gestión de inventario y ventas. 
Desde una perspectiva técnica, el sistema interactúa con los siguientes elementos: 
●​ Usuario principal, quien opera directamente la aplicación 
●​ Servicios de almacenamiento en la nube, utilizados para respaldo y recuperación de 
datos 
 
 
●​ Dispositivo de escaneo (cámara), empleado para la lectura de códigos de barras 
El sistema gestiona la información de manera local y ejecuta sus operaciones principales sin 
dependencia continua de servicios externos, limitando dichas integraciones a 
funcionalidades específicas. 
2.2 Funciones generales del sistema 
●​ El sistema proporciona las siguientes funciones a nivel general: 
●​ Gestionar productos y categorías del inventario 
●​ Registrar y controlar lotes con fechas de vencimiento 
●​ Procesar ventas al contado y a crédito 
●​ Administrar clientes y sus deudas 
●​ Generar boletas de venta y reportes 
●​ Emitir notificaciones sobre eventos críticos del inventario 
●​ Permitir búsqueda, filtrado y consulta de información 
●​ Exportar información en formato PDF 
2.3 Tipos de usuarios 
El sistema contempla los siguientes tipos de actores: 
●​ Usuario principal (Propietaria) 
Actor humano responsable de ejecutar todas las funcionalidades del sistema, 
incluyendo gestión de inventario, ventas, clientes y reportes. 
●​ Sistema externo (Servicios en la nube) 
Actor no humano que interviene en los procesos de respaldo y recuperación de 
información. 
●​ Dispositivo de escaneo 
Componente del sistema que permite capturar códigos de barras mediante la cámara 
del dispositivo. 
2.4 Suposiciones y dependencias 
●​ El usuario dispone de un dispositivo móvil compatible. 
●​ El usuario tiene acceso a un correo electrónico válido para el registro. 
●​ El sistema puede acceder a la cámara del dispositivo para el escaneo de códigos de 
barras. 
●​ El almacenamiento local del dispositivo es suficiente para operar el sistema. 
 
 
●​ La conexión a internet es necesaria únicamente para procesos de verificación de 
cuenta y respaldo de información.

CU-01 
Agregar Productos 
Descripción 
Permite al usuario ingresar nuevos artículos al catálogo del sistema. 
Actor 
Usuario 
Precondiciones 
El usuario debe haber iniciado sesión y estar en la sección de Inventario. 
 
 
Postcondiciones 
El nuevo producto queda registrado en la base de datos. 
Flujo principal 
1. El usuario selecciona "Agregar Producto". 
2. El sistema solicita datos (nombre, precio, stock inicial). 
3. El usuario ingresa la información. 
4. El sistema valida los datos y guarda el registro. 
Excepciones 
- El producto ya existe (el sistema notifica duplicidad). 
- Datos incompletos o formato de precio incorrecto. 
 
CU-02 
Actualizar Stock 
Descripción 
Permite modificar la cantidad disponible de un producto existente. 
Actor 
Usuario 
Precondiciones 
El producto debe estar previamente registrado. 
Postcondiciones 
Se actualiza la cantidad de existencias en el sistema. 
Flujo principal 
1. El usuario busca el producto. 
2. El sistema muestra los datos actuales. 
3. El usuario ingresa la nueva cantidad o el monto a sumar. 
4. El sistema actualiza el registro. 
Excepciones 
- Producto no encontrado. 
- Cantidad ingresada negativa (no permitida). 
 
CU-03 
Eliminar Productos 
Descripción 
Permite retirar un producto del catálogo de forma definitiva. 
Actor 
Usuario 
Precondiciones 
El producto debe existir y no tener ventas pendientes relacionadas. 
 
 
Postcondiciones 
El registro del producto es eliminado o marcado como inactivo. 
Flujo principal 
1. El usuario selecciona un producto de la lista. 
2. El usuario elige la opción "Eliminar Producto". 
3. El sistema solicita una confirmación de seguridad. 
4. El usuario confirma la acción. 
5. El sistema elimina el registro y actualiza la lista. 
Excepciones 
- El usuario cancela la confirmación. 
- El producto no puede eliminarse por tener historial de ventas (integridad 
referencial). 
 
 
CU-04 
Ver Inventario de productos 
Descripción 
Visualización de la lista de productos 
Actor 
Usuario 
Precondiciones 
Existencia de productos en la base de datos. 
Postcondiciones 
El usuario visualiza la información y opcionalmente se genera un archivo de 
reporte. 
Flujo principal 
1. El usuario accede a "Ver Inventario". 
2. El sistema despliega la lista. 
3. El usuario visualiza la lista de productos 
Excepciones 
- No hay productos para mostrar. 
 
 
CU-05 
Generar Reporte de Inventario 
Descripción 
Generación de un reporte detallado de Inventario 
Actor 
Usuario 
 
 
Precondiciones 
Existencia de productos en la base de datos. 
Postcondiciones 
El sistema genera un archivo de reporte. 
Flujo principal 
1. El usuario accede a "Ver Inventario". 
2. El sistema despliega la lista. 
3. El usuario solicita "Generar reporte de inventario". 
4. El sistema exporta la lista. 
Excepciones 
- No hay productos para mostrar. 
- Error al generar el archivo de reporte. 
 
 
CU-06 
Escanear productos 
Descripción 
Proceso de preparación de una venta mediante el escaneo de productos. 
Actor 
Usuario 
Precondiciones 
Los productos deben tener código de barras o estar en sistema. 
Postcondiciones 
Se genera una lista temporal de productos para la venta. 
Flujo principal

6.1 Tecnológicas 
●​ El sistema se desarrolla utilizando Flutter como framework principal.

6.3 De diseño 
●​ La interfaz prioriza simplicidad sobre complejidad funcional. 
●​ Se limita la personalización para mantener consistencia en la interacción.
